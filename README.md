# MRPG Overwatch

**MRPG Overwatch** — это кастомный RPG-режим для Overwatch с упором на PvP, глубокую систему прокачки и экономику. 
Проект разрабатывается с помощью Python-подобного языка и транслируется в код Overwatch Workshop через расширение [OverPy](https://github.com/Zezombye/overpy).

## Описание проекта
В режиме игроки начинают в безопасной зоне (спавн) и выходят в открытый мир для фарма опыта, денег и артефактов. Главная особенность режима — разнообразие путей прогрессии. Игроки могут развиваться через стандартные классы, механики "ритуального самоубийства" (Вампиризм) или экономику (Артефакты и Изумруды). 
Особое внимание уделяется балансу: грамотный геймдизайн позволяет новичкам комфортно вливаться в игру даже на поздних этапах существования лобби.

## Структура репозитория

- `main.opy` — точка входа в проект.
- `settings/` — конфигурация массивов, константы и настройки лобби.
- `variables/` — объявление глобальных переменных и переменных игроков.
- `core/` — базовые системы (инициализация, HUD, базовый левелинг).
- `classes/` — реализация способностей и логики классов (например, Читатель).
- `systems/` — глобальные системы инфраструктуры (жизненный цикл баффов/дебаффов).
- `docs/` — проектная документация (геймдизайн).

## Документация для разработчиков

Проект имеет строгие рамки по оптимизации из-за ограничений движка Overwatch Workshop (максимум 256 эффектов, 128 текстов и ограниченная серверная память).

* **[BUILD.md](BUILD.md)** — **читается первым при любой задаче.** Чек-листы «что обязан сделать, добавляя бафф / стат / способность / зону», чтобы не оставить баг.
* **[ARCHITECTURE.md](ARCHITECTURE.md)** — живая карта реализованных систем и известных швов. Контекст для нового диалога с нейросетью.
* **[Геймдизайн и Баланс](docs/gamedesign.md)** — концепция режима, механики прокачки, экономика и психология удержания игроков (GDD).
* **[Архитектура и Правила Кода](CODESTYLE.md)** — стандарты написания OverPy кода, работа с массивами, инфраструктурой эффектов и обход багов движка.
* **Гайдлайны для ИИ** — базовый контекст для работы нейросетей описан в 5-м разделе файла `CODESTYLE.md`.

## Проверка перед сборкой

Перед компиляцией запускай статический валидатор связей между системами:
```bash
python tools/validate.py
```
`0 error(s)` — можно собирать. Есть ERROR — что-то забыли связать (см. адрес `файл:строка` в выводе). Расшифровка проверок — в `ARCHITECTURE.md` и в шапке `tools/validate.py`.

## Инструкция по сборке

1. Установите [OverPy CLI](https://github.com/Zezombye/overpy/wiki/Installation).
2. Запустите компиляцию:
   ```bash
   overpy main.opy
   ```
3. Сгенерированный код будет скопирован в ваш буфер обмена.
4. Откройте Overwatch -> Custom Games -> Create -> Settings -> Workshop -> Paste.
5. Наслаждайтесь игрой! Убедитесь, что настройки лобби соответствуют требованиям (см. раздел 7 в [CODESTYLE.md](CODESTYLE.md)).
