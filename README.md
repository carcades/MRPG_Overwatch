Данный проект описывает кастомный режим для Overwatch благодаря трансляции 
python-подобного кода в код Overwatch Workshop через расширение OverPy.

Структура режима:
режим представляет собой rpg-подобную игру с упором на pvp.
- игрок появляется в безопасной зоне на спавне
- на спавне расположены зоны в которых он может прокачать характеристики, 
либо войти в другие локации
- есть механика изумрудов, которые работают по принципу биткоинов:
чем больше их существует на сервере, тем они дороже
- есть механика области с опытом, которая увеличивает уровень, но уменьшает выдачу
засчет большего кол-ва людей внутри
- есть механика получения денег через область которая наносит дамаг в процентах
- есть механика сакрального алтаря или как-то так, место где человек выбирает 
новый класс и прокачивает способности текущего
- есть механика сепукко, которая позволяет получать уникальные способности через
самоубийство и сброс навыков
- есть механики прокачки, которые включают в себя:
1) базовая прокачка уровня доступная всегда - игрок идет к зоне опыта
и, конкурируя с другими игроками получает опыт (чем меньше игроков
в зоне, тем быстрее получается опыт), максимальное значение прокачиваемых 
характеристик ограничено тиром персонажа
2) механика классов - (то, что определяет тир), когда игрок накапливает
достаточно уровней, он может взять класс, классы задумываются обычные и секретные,
новый класс повышает тир пользователя, увеличивая силу способностей и максимальное
значение характеристик, обычно новый класс сбрасывает предыдущие характеристики, у
классов есть линии, например на первом тире человек условно читатель молитв, на
втором какой-нибудь благословленный воин и, после выбора первого класса, человек
может выбрать следующие классы только из той же ветки.
3) механика вампиризма - человек качается до определенного уровня и делает сеппуку,
с этим его характеристики сбрасываются и он получает доступ к пулу особенных
blood способностей, которые можно приобрести только за очки крови, здесь задумывается
реализация секретного класса.
4) механика богатства - человек может накопить деньги, и пойти в магазин артефактов,
чтобы купить себе артефакт, который не будет сбрасываться при переходе между классами,
и, который будет давать бонусы к хп, защите, или будет давать уникальные замысловатые
способности. тут также может быть секретный класс, который открывается при накоплении
n золота

Требования к проекту:
1. не перегрузить сервер. 8 игроков должны иметь возможность играть без перегруза
2. соблюдать лимиты сущностей и визуальных эффектов. смотря на мои монструозные планы
балансировать на грани захватывающей игры, например самое главное обращать внимание на
следующие ограничения:
- максимум визуальных эффектов 256 (у меня визуальные эффекты повсюду, 
как статичные на спавне, так и динамические в бою)
- максимум 128 текстовых элементов
- максимальная задержка сервера 250, сложные операции должны быть оптимизированны
для того чтобы бороться с ограниченным кол-вом эффектов и текста есть лазейка - 
все элементы кроме динамических (в бою), должны создаваться один раз глобально,
итого при построении архитектуры проекта нужно создавать минимальное количество уникальных
объектов текста и визуальных эффектов, пытаясь впихнуть все в один
3. быть читаемым человеком. в этом я надеюсь мне поможет OverPy

Цели проекта:
1. создать интересный rpg-режим в который можно будет играть часами,
разными способами получать силу, использовать разные стратегии и линии классов.
2. удерживать онлайн. даже самый интересный pvp режим для меня одного, будет
скучным если в него играть без людей. новичкам должно быть интересно играть
в лобби даже спустя час после старта, когда остальные игроки перекачаны. пока 
я вижу несколько причин, по которым новички не хотят играть в старых лобби в подобных
режимах:
- слишком сильный перекач старых игроков, так что новые игроки даже в 11 человек
не смогут убить старого. в моем проекте задумано чтобы повышение классов давало
интересные сбалансированные способности, которые все же могут быть побеждены
новым игроком при нужном скилле.
- слишком сильная конкуренция за места прокачки. в моем проекте я планирую сделать 
несколько возможных стратегий прокачки, если старые мощные игроки заняли одни места,
то новички могут пойти по другим путям.
- недостаточно целей для развития. игроки могут просто взять и не захотеть драться ради 
дальнейшей прокачки, тем самым убивая самый главный драйвер интереса в наборе силы и в
самой игре. ведь если игроки не дерутся, то зачем становиться сильнее? есть мысль о 
введении pve ботов, которые могут представлять интерес для истребления и быть конечной 
целью прокачки, о реализации пока мыслей нет.

Возможности:
1) стандартные возможности Overwatch Workshop за исключением того что убрано расширениями
2) 8 игроков в лобби
3) максимум ботов в слоте игрока 16
4) включены лучевые эффекты
5) включены лучевые звуки
6) выключены звуки баффов и дебаффов
7) включены эффекты баффов статуса
8) выключены эффекты дебаффов статуса
9) выключены эффекты энергетических взрывов
10) выключены звуки взрывов
11) включены кинетические эффекты взрывов
12) включен пункт play more effects
13) включены снаряды (projectiles)
14) включен spawn more dummy bots на 12
16) минимальная задержка во времени 0.016 сек
17) максимум переменных игрока 128
18) максимум глобальных переменных 128
19) максимальный размер массива - 1000 элементов 

Подход к реализации:
сначала должны быть сделаны основные фичи которые определяют ценность проекта,
потом уже второстепенные замысловатые механики.

Инструкция для нейросети:
- не менять ничего в коде без прямого указания
